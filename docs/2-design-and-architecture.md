# Задание 2: Проектиране и архитектура на решението

## 1. Тип и структура на приложението

### Тип на приложението
Уеб приложение (Web Application), достъпно през браузър.

### Организация на проекта
Проектът е структуриран по слоеве (layered architecture):

- Frontend (потребителски интерфейс)
- Backend (бизнес логика)
- Database (съхранение на данни)

### Основни модули / папки

- `/client` – frontend част
- `/server` – backend логика
- `/models` – модели на данни
- `/routes` – API маршрути
- `/controllers` – логика за обработка
- `/services` – бизнес логика


---

## 2. Модел на данните

### Основни ентитита

#### User
- id
- username
- email
- password
- role

#### Post
- id
- title
- content
- createdAt
- userId

#### Comment
- id
- content
- postId
- userId

### Връзки

- Един User има много Post-ове (1:N)
- Един Post има много Comment-и (1:N)
- Един Comment принадлежи към един User и един Post

## 3. Диаграма или структурирано описание

### Таблица с основни класове

| Клас       | Роля                         |

## 4. Потребителски поток

### Основни страници

- Начална страница
- Страница за вход/регистрация
- Профил
- Списък с публикации
- Създаване на публикация

### Поток

1. Потребителят отваря началната страница
2. Регистрира се или влиза
3. Преглежда публикации
4. Създава нова публикация
5. Редактира или изтрива свои публикации

### Основни действия

- регистрация
- вход
- създаване на съдържание
- редакция
- изтриване

## 5. Използвани технологии

### Езици
- JavaScript

### Framework
- Node.js / Express (backend)
- React (frontend)

### База данни
- MongoDB

### Библиотеки
- Mongoose (ORM)
- JWT (автентикация)
- bcrypt (хеширане на пароли)

### Други технологии
- Git (version control)

### Защо тези технологии

- JavaScript позволява използване на един език за frontend и backend
- Express е лек и подходящ за API
- React осигурява динамичен интерфейс
- MongoDB е гъвкава база данни за този тип проект
